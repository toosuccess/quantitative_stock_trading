# 消息面和政策面评分调整计划

## 一、需求分析

### 1.1 用户需求
1. **消息面和政策面评分调整**：不需要基准分，直接根据消息的利好利空进行加减分
2. **评分详情页面修复**：目前各维度的内容都是空的，需要检查并修复
3. **股票池列表评分调整**：各维度评分显示需要做相应调整

### 1.2 当前实现问题

#### 问题1：消息面和政策面基准分问题
- 当前实现使用50分作为基准分
- 前端显示"注：基准分50分，高于50加分，低于50扣分"
- 用户要求：直接根据利好利空加减分，不需要基准分

#### 问题2：评分详情页面数据为空
- 各Tab页（技术面、基本面、消息面、政策面、减项）都显示"暂无数据"
- 原因分析：
  - 数据库中明细字段可能为空
  - 前端解析逻辑可能有问题
  - 后端API返回数据不完整

## 三、详细修改方案

### 3.1 消息面评分逻辑修改

**修改前**：
```python
score = 50  # 基准分
# 利好消息 +20，利空消息 -20
```

**修改后**：
```python
score = 0  # 从0分开始
# 利好消息直接加分，利空消息直接扣分
# 最终得分范围：-50 到 +50
```

### 3.2 政策面评分逻辑修改

**修改前**：
```python
score = 50  # 基准分
# 根据政策支持程度调整
```

**修改后**：
```python
score = 0  # 从0分开始
# 利好政策直接加分，利空政策直接扣分
# 最终得分范围：-30 到 +30
```

### 3.3 综合评分公式调整

**修改前**：
```
综合评分 = 技术面×0.5 + 基本面×0.5 + (消息面-50)×0.2 + (政策面-50)×0.2 - 减项扣分
```

**修改后**：
```
综合评分 = 技术面×0.5 + 基本面×0.5 + 消息面加减分 + 政策面加减分 - 减项扣分
```

### 3.4 前端显示调整

1. **ScoreDetail.vue**：
   - 移除"注：基准分50分，高于50加分，低于50扣分"文字
   - 调整消息面和政策面评分显示，支持负分显示
   - 修复明细数据为空的问题

2. **StockSelection.vue**：
   - 调整消息面和政策面评分显示逻辑
   - 支持正负分显示

### 3.5 评分详情数据修复

检查并修复以下问题：
1. 数据库中 technical_detail, fundamental_detail 等字段是否有数据
2. 后端API是否正确返回明细数据
3. 前端解析JSON是否正确

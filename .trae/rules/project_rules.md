# 6A5S AI编程规则（Trae专用）
## 6A工作流强制要求
1. Align（需求对齐）：所有开发前必须输出《需求对齐文档》，明确功能边界与Out of Scope
2. Architecture（架构设计）：输出《架构设计文档》+ 核心流程图（Mermaid格式）
3. Atomize（任务量化）：拆分原子任务，每个任务不超过8小时工作量，含验收标准
4. Approve（人工审批）：架构设计、任务拆分、核心模块代码必须经人工确认后执行
5. Act（执行落地）：按任务顺序开发，每完成1个原子任务先本地验证（编译+单元测试）
6. Assess（评估验收）：输出《验收文档》，含功能测试报告、性能指标、安全检查结果

## 5S约束规则
1. Systematize（文档管理）：代码修改必须同步更新对应文档，无文档不提交
2. Sequential（流程规范）：严格按“对齐→设计→拆分→审批→执行→评估”顺序，不跳步
3. Solve（问题解决）：优先查阅官方文档/项目历史解决方案，再向AI提问，附参考链接
4. Strict（执行约束）：禁止无测试提交、禁止硬编码敏感信息、禁止违反编码规范
5. Standardize（输出标准）：
   - 编码规范：Python遵循PEP8，JavaScript遵循Airbnb规范
   - 依赖管理：自动生成requirements.txt/package.json并锁定版本
   - 测试要求：核心功能单元测试覆盖率≥80%

## 测试要求
1. 编码结束后，根据修改内容更新测试用例，确保测试覆盖率≥80%，并自动运行测试
2. 测试用例必须覆盖正常场景、边界场景、异常场景
3. 测试代码必须符合编码规范，注释清晰
4. 请使用真实的用户测试，模拟真实场景，确保功能正常
   -东华软件002065
   -茅台600519 龙头股
5.每次修改代码后，都要规划测试方案，测试用例（测试用例包括页面功能、接口功能、性能测试等），并执行测试，输出测试报告。
6.每次功能修改，都要端到端的从页面发起测试，使用playwright进行测试。
7.完成测试后，自动删除测试文件，不需要征求开发者确认。



## 授权规则
1.大模型向开发者，请求Invoke-WebRequest授权时，默认同意授权
2.测试中你自己创建的文件，直接删除就行，不需要征求开发者确认。
3.plan模式生成的执行方案，在执行完后，自动删除，不需要征求开发者确认。

## plan模式
 1.计划模式，都要询问开发者确认。允许开发者，在对话中修改计划。


## 指令要求
1.如果我说，帮我选股，需要调用选股技能：stock_selector_skills.py ，并能完成评价。
  1）选股技能，是根据十五五规划，粗粒的选股，动态扩充核心股票池中，排除掉ST股。选股技能输出股票存储到stock_basic_info （股票基本信息）。

2.如果我说，帮我评价某股票，需要调用评价技能：stock_evaluator_skills.py，并能完成评价，评价结果需要输出到数据库中。
   1）评价是对数据库stock_basic_info表中记录的股票进行评价.
   2）如果股票不在数据库中，需要先添加到数据库中，再进行评价。
   3）评分记录表（score_record）中，需要记录股票的评价结果，包括股票代码、评价时间、评价结果等。
   4）评价技能，是对数据库表stock_basic_info中记录的股票，按照评价策略进行打分，打分结果记录到score_record （评分记录）

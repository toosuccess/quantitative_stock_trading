# 交易计划查看和执行步骤数量限制计划

## 需求分析

### 问题1：已完成交易计划的查看按钮
**当前情况**：
- TradePlan.vue中"查看执行"按钮只在executing状态显示
- completed状态的交易计划没有查看按钮

**期望**：
- 对已完成的交易计划，提供"查看"按钮
- 点击后跳转到执行步骤列表

### 问题2：添加执行步骤时的数量限制
**当前情况**：
- 添加执行步骤时只显示计划数量输入框
- 没有显示持仓数量和剩余数量
- 没有数量限制

**期望**：
- 显示计划数量、持仓数量、剩余数量
- 加仓时：不能超过剩余数量
- 减仓时：不能超过持仓数量

## 实施方案

### 问题1修复：添加查看按钮

文件：`frontend/src/views/TradePlan.vue`

```vue
<el-button 
  v-if="scope.row.status === 'executing' || scope.row.status === 'completed'"
  size="small" 
  type="warning" 
  @click="viewExecution(scope.row)"
>
  查看
</el-button>
```

### 问题2修复：添加数量显示和限制

文件：`frontend/src/views/TradeExecution.vue`

1. 添加数量显示：
```vue
<el-form-item label="计划数量">
  {{ selectedPlan?.planned_quantity || 0 }}
</el-form-item>
<el-form-item label="持仓数量">
  {{ selectedPlan?.holding_quantity || 0 }}
</el-form-item>
<el-form-item label="剩余数量">
  {{ remainingQuantity }}
</el-form-item>
```

2. 添加数量限制：
```vue
<el-form-item label="数量">
  <el-input-number 
    v-model="addStepForm.planned_quantity" 
    :min="0" 
    :max="maxQuantity"
    :step="100" 
  />
</el-form-item>
```

3. 添加计算属性：
```javascript
const selectedPlan = computed(() => {
  return allPlans.value.find(p => p.plan_id === addStepForm.value.plan_id)
})

const remainingQuantity = computed(() => {
  const plan = selectedPlan.value
  if (!plan) return 0
  return plan.planned_quantity - (plan.holding_quantity || 0)
})

const maxQuantity = computed(() => {
  const direction = addStepForm.value.trade_direction
  if (direction === '加仓') {
    return remainingQuantity.value
  } else if (direction === '减仓') {
    return selectedPlan.value?.holding_quantity || 0
  }
  return 0
})
```

# 页面优化计划

## 需求分析

### 1. 选股结果 → 股票池

* 将"选股结果"页面改为"股票池"页面

* 点击股票可查看该股票的评分详情

### 2. 评分结果界面优化

* 一只股票会有多次评分记录

* 支持查看同一股票的多次评分历史

* 显示各指标的变化趋势图

### 3. 交易执行界面优化

* 添加查询条件：交易计划、股票代码

* 已执行和待执行合并显示（不分页签）

* 支持筛选和搜索

***

## 实施方案

### 一、股票池页面改造 (StockSelection.vue → StockPool.vue)

#### 页面功能

1. 显示所有已评分股票列表（去重）
2. 每只股票显示最新评分、评级、评分次数
3. 点击股票跳转到评分详情页

#### 数据结构

```javascript
// 股票池数据
{
  stock_code: '600410',
  stock_name: '华胜天成',
  latest_score: 100.0,
  latest_rating: '强烈推荐',
  score_count: 3,  // 评分次数
  latest_date: '2026-02-18'
}
```

#### 后端API调整

* 新增 `GET /api/v1/stocks/pool/scores` - 获取股票池评分汇总

### 二、评分详情页优化 (Score.vue)

#### 页面功能

1. 显示股票基本信息
2. 显示评分历史列表（按日期倒序）
3. 显示技术指标变化趋势图（ECharts）

#### 数据结构

```javascript
// 评分历史
{
  score_date: '2026-02-18',
  total_score: 100,
  ma_score: 25,
  macd_score: 20,
  rsi_score: 0,
  bollinger_score: 15,
  volume_score: 25,
  obv_score: 15,
  close_price: 24.61
}
```

#### 后端API调整

* 修改 `GET /api/v1/scores/{code}` - 返回该股票所有评分记录

### 三、交易执行页面优化 (TradeExecution.vue)

#### 页面功能

1. 添加筛选条件：

   * 交易计划下拉选择

   * 股票代码输入框

   * 状态选择（全部/待执行/已执行）
2. 合并显示所有执行步骤
3. 支持执行操作

#### 后端API调整

* 修改 `GET /api/v1/trade/plans` - 支持筛选参数

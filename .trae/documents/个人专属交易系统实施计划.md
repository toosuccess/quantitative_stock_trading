# 个人专属交易系统实施计划

## 一、项目概述

### 1.1 核心目标
打造一个"去情绪化、可量化、可追溯、可重复、可迭代"的交易系统，通过标准化流程解决炒股核心痛点，最终实现稳定盈利。

### 1.2 技术架构
| 层次 | 技术选型 | 说明 |
| :--- | :--- | :--- |
| 数据源 | AkShare / Tushare / QMT | 免费且成熟的金融数据开源库 |
| 后端逻辑层 | Python (FastAPI / Flask) | 金融工程首选，库生态丰富 |
| 数据存储 | SQLite | 简单、轻量级，适合个人使用 |
| 前端看板 | Vue 3 + ECharts | K线图、评分饼图、资产曲线可视化 |
| 任务调度 | APScheduler | 定时获取收盘数据、自动计算评分 |

## 二、数据库设计（已完成）

### 2.1 数据库表结构
已设计7张核心表：
1. **account_info** - 账户信息表
2. **stock_basic_info** - 股票基本信息表
3. **score_record** - 评分记录表（含选股原因：基本面+技术面）
4. **trade_plan** - 交易计划表（引用评分记录，记录盈亏）
5. **trade_execution_step** - 交易执行步骤表
6. **trade_record** - 交易记录表
7. **review_record** - 复盘记录表

### 2.2 实体关系
```
账户信息 ──1:N──> 交易记录
股票基本信息 ──1:N──> 评分记录
股票基本信息 ──1:N──> 交易计划
交易计划 ──1:N──> 交易执行步骤
交易计划 ──1:N──> 复盘记录
交易执行步骤 ──N:1──> 交易记录
```

## 三、核心功能模块

### 3.1 选股模块（含评分模块）
- **输入**：技术指标和基本面数据
- **处理**：根据选股策略筛选符合条件的股票
- **输出**：评分结果存入 score_record 表

**技术分析指标要求**：
- 均线系统：股价站稳20日均线且M20向上；60/180/250日均线多头排列
- 成交量：当日成交量＞5日/60日均量；换手率5%-20%
- 趋势指标：DMI的+DI＞-DI且ADX/ADXR上升；**MACD指标：DIFF＞DEA>0**
- 资金指标：OBV＞MAOBV且持续上行，资金净流入
- 布林线：股价位于布林线的上轨和中轨之间，且未超买

### 3.2 交易计划模块
- **标准化表单**：入场必须满足"计划先行"
- **入场逻辑**：记录买入理由（基于评分结果）
- **仓位管理**：根据个股波动率（ATR）自动计算建议仓位
- **退出策略**：明确止损价、止盈价、时间止损

### 3.3 交易执行模块
- **定时执行**：收盘前15分钟，根据交易计划执行交易
- **手工执行**：界面提供操作入口，支持手工执行交易计划
- **记录存储**：交易记录存入 trade_record 表

### 3.4 复盘模块
- **记录与匹配**：自动将实际成交数据与预设交易计划进行匹配
- **情绪监控**：用户需在成交后记录当时的心理状态
- **失败分类**：系统引导用户将亏损归因分析
- **统计看板**：胜率、盈亏比、最大回撤、夏普比率可视化

### 3.5 模拟操作模块
- **虚拟账户**：支持多策略同时运行（A/B Test）
- **模拟撮合**：模拟真实买卖单、手续费、滑点
- **策略回测**：支持对选股和评分模型进行历史回测

## 四、用户旅程

### 4.1 选股流程
1. 运行选股技能，根据策略筛选股票
2. 数据存入 stock_basic_info 和 score_record 表

### 4.2 评分流程
1. 运行评分技能，计算股票评分
2. 评分策略与选股策略保持一致
3. 数据存入 score_record 表

### 4.3 交易计划流程
1. 在界面上查看选股结果和评分结果
2. 根据策略填写交易计划
3. 数据存入 trade_plan 和 trade_execution_step 表

### 4.4 交易执行流程
1. 定时任务在收盘前15分钟执行交易
2. 或通过界面手工执行交易计划
3. 交易记录存入 trade_record 表

### 4.5 复盘分析流程
1. 根据交易记录分析交易结果
2. 计算胜率、盈亏比、最大回撤、夏普比率
3. 运行复盘分析技能，分析交易结果

# 评分技能 Spec

## Why
当前系统已有选股技能，可以筛选股票并存入数据库。但缺少一个独立的评分技能，用于对数据库中已存在的股票池股票进行定期评分，以便跟踪股票的技术面变化和投资价值变化。

## What Changes
- 新增评分技能Python模块
- 实现从数据库读取股票池股票
- 实现技术面评分计算
- 实现评分记录存储到数据库

## Impact
- 新增文件：`skills/stock_scoring_skill.py`
- 数据库：写入`score_record`表
- 依赖：复用选股技能的评分逻辑

## ADDED Requirements

### Requirement: 股票池读取
系统应从数据库中读取需要评分的股票列表。

#### Scenario: 读取股票池
- **WHEN** 用户调用评分技能
- **THEN** 系统从`stock_basic_info`表或`score_record`表读取股票代码列表

### Requirement: 技术面评分
系统应按照选股技能的评价策略计算技术面评分。

#### Scenario: 计算技术指标
- **WHEN** 系统获取股票K线数据
- **THEN** 计算均线系统、成交量、趋势指标、资金指标、布林线等指标

#### Scenario: 计算技术面得分
- **WHEN** 所有技术指标计算完成
- **THEN** 根据评分规则计算各指标得分和总得分

### Requirement: 评分记录存储
系统应将评分结果存储到数据库。

#### Scenario: 存储评分记录
- **WHEN** 评分计算完成
- **THEN** 将评分记录插入`score_record`表
- **AND** 如果当天已存在该股票评分，则更新记录

### Requirement: 评分规则
系统应按照以下规则计算技术面得分：

| 指标 | 条件 | 得分 |
|------|------|------|
| 均线系统 | 股价站稳20日均线且M20向上；60/180/250日均线多头排列 | 25分 |
| 成交量 | 当日成交量＞5日/60日均量；换手率5%-20% | 25分 |
| 趋势指标 | DMI的+DI＞-DI且ADX/ADXR上升；MACD: DIFF＞DEA>0 | 20分 |
| 资金指标 | OBV＞MAOBV且持续上行，资金净流入 | 15分 |
| 布林线 | 股价位于布林线的上轨和中轨之间，且未超买 | 15分 |

### Requirement: 评级输出
系统应根据综合得分输出评级。

| 得分范围 | 评级 |
|----------|------|
| 90-100分 | 强烈推荐 |
| 70-89分 | 推荐 |
| 50-69分 | 中性 |
| 30-49分 | 观望 |
| 0-29分 | 不推荐 |

# 清仓逻辑检查和修正计划

## 需求分析

### 当前情况
当前逻辑：
1. `止损清仓`、`清仓`、`止盈清仓`：直接标记计划完成
2. `减仓`：检查持仓数量，如果<=0则标记完成
3. 其他操作：检查所有步骤是否完成

### 问题
- `止盈减仓`没有处理
- 应该对所有卖出操作都检查持仓数量

### 期望
以下操作如果卖出股数=持仓股数，都代表清仓：
- 减仓
- 止盈减仓
- 止损清仓
- 清仓
- 止盈清仓

---

## 实施方案

### 修改逻辑

文件：`backend/app/services/trading_manager.py`

修改 `execute_step` 方法：

```python
# 原来：分开处理
if step['trade_direction'] in ['止损清仓', '清仓', '止盈清仓']:
    self.update_trade_plan(step['plan_id'], {'status': 'completed'})
elif step['trade_direction'] == '减仓':
    holding_info = self.get_holding_profit(step['plan_id'], 0)
    current_holding = holding_info.get('holding_quantity', 0)
    if current_holding <= 0:
        self.update_trade_plan(step['plan_id'], {'status': 'completed'})

# 修改为：统一处理所有卖出操作
if step['trade_direction'] in ['减仓', '止盈减仓', '止损清仓', '清仓', '止盈清仓']:
    holding_info = self.get_holding_profit(step['plan_id'], 0)
    current_holding = holding_info.get('holding_quantity', 0)
    if current_holding <= 0:
        self.update_trade_plan(step['plan_id'], {'status': 'completed'})
```

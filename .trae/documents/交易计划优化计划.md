# 交易计划优化计划

## 需求分析

### 1. 交易计划删除功能

* 当前状态：删除按钮存在但功能未实现

* 需求：实现交易计划的删除功能

### 2. 交易计划执行流程优化

* 当前状态：执行按钮只是弹出对话框，没有状态变更

* 需求：

  * 点击执行后，计划状态变为"执行中"

  * 自动初始化交易执行步骤（建仓、加仓、减仓、清仓）

  * 从交易计划列表可跳转到交易执行页面

***

## 实施方案

### 一、交易计划删除功能

#### 后端API

* 已有接口：`DELETE /api/v1/trade/plan/{plan_id}`

* 需要实现：TradingManager中的删除方法

#### 前端实现

* TradePlan.vue中调用删除API

* 删除成功后刷新列表

### 二、交易计划执行流程

#### 执行步骤设计

```
交易计划（待执行）
    ↓ 点击执行
状态变为"执行中" + 自动生成执行步骤
    ↓
交易执行步骤表（trade_execution_step）
    - 建仓步骤（目标价格、计划数量）
    - 加仓步骤（可选）
    - 减仓步骤（可选）
    - 清仓步骤（止盈/止损触发）
```

#### 后端API调整

1. 新增 `POST /api/v1/trade/plan/{plan_id}/execute` - 执行计划

   * 更新计划状态为"执行中"

   * 自动创建执行步骤

2. 新增 `GET /api/v1/trade/steps` - 获取执行步骤列表

   * 支持按计划ID筛选

#### 执行步骤数据结构

```javascript
{
  step_id: 'STEP_xxx',
  plan_id: 'PLAN_xxx',
  stock_code: '600410',
  stock_name: '华胜天成',
  trade_direction: '建仓',  // 建仓/加仓/减仓/清仓
  target_price: 9.50,
  planned_quantity: 1000,
  executed_quantity: 0,
  remaining_quantity: 1000,
  status: 'pending',
  planned_date: '2026-02-18'
}
```

### 三、页面跳转优化

#### TradePlan.vue

* 执行按钮：调用执行API，成功后跳转到交易执行页

* 查看执行按钮：状态为"执行中"时显示，跳转到交易执行页

#### TradeExecution.vue

* 显示执行步骤列表

* 支持按计划ID筛选

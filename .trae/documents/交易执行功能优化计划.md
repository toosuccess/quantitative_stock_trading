# 交易执行功能优化计划

## 需求分析

### 问题1：计划编号格式
**当前格式**：`PLAN_f05ed4f9`
**期望格式**：`PLAN_时间戳_股票名称`，如 `PLAN_20260218230000_华胜天成`

### 问题2：添加"止盈清仓"交易方向
**当前情况**：
- 执行计划时只创建"止损清仓"步骤
- 需要添加"止盈清仓"步骤

### 问题3：去掉"止损清仓"选项
**当前情况**：
- 添加执行计划弹窗中有"止损清仓"选项
- 用户要求去掉这个选项

### 问题4：止损清仓成交数量等于持仓数量
**当前情况**：
- 止损清仓时成交数量默认为0
- 期望：成交数量自动设置为持仓数量

### 问题5：添加"持仓数量"显示
**当前情况**：
- 执行步骤弹窗中没有显示持仓数量
- 需要在弹窗中添加持仓数量显示

### 问题6：止损清仓成交数量验证问题
**当前情况**：
- 输入成交数量后，点击确认执行，系统提醒还要输入成交数量
- 然后成交数量变成0

## 实施方案

### 问题1修复：计划编号格式

文件：`backend/app/services/trading_manager.py`

```python
def create_trade_plan(self, plan_data: Dict) -> str:
    timestamp = datetime.now().strftime('%Y%m%d%H%M%S')
    stock_name = plan_data.get('stock_name', '')
    plan_id = f"PLAN_{timestamp}_{stock_name}"
```

### 问题2修复：添加"止盈清仓"步骤

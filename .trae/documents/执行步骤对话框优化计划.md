# 执行步骤对话框优化计划

## 需求分析

### 1. 建仓目标价格问题

**当前逻辑**：建仓步骤的target\_price = stop\_loss\_price（止损价格）
**期望逻辑**：建仓步骤的target\_price = 初始价格（创建计划时的当前价格）

### 2. 去掉已执行数量

**当前**：对话框显示"已执行数量"字段
**期望**：移除此字段

### 3. 剩余数量计算

**当前**：剩余数量 = 计划数量 - 已执行数量
**期望**：剩余数量 = 计划数量 - 当前持仓数量（需要从trade\_record计算）

### 4. 成交价格默认值

**当前**：成交价格 = target\_price（目标价格）

**期望**：成交价格 = 当前实时价格（从score\_record获取），需要检查别的页面，如果目标价格为0的，需要使用当前价格。

### 5. 加仓数量

**当前**：加仓数量 = planned\_quantity（用户手动输入）
**期望**：加仓数量 = 建仓数量（默认值）

### 6. 减仓逻辑

**当前**：减仓数量 = planned\_quantity（用户手动输入）
**期望**：减仓数量 = 建仓数量（默认值），最大不超过持仓数量

```python
def get_holding_quantity(self, plan_id: str) -> int:
    """获取计划当前持仓数量"""
    cursor = self.conn.cursor()
    cursor.execute('''
        SELECT 
            COALESCE(SUM(CASE WHEN trade_type = '买入' THEN trade_quantity ELSE 0 END), 0) -
            COALESCE(SUM(CASE WHEN trade_type = '卖出' THEN trade_quantity ELSE 0 END), 0) as holding
        FROM trade_record 
        WHERE plan_id = ?
    ''', (plan_id,))
    row = cursor.fetchone()
    return row['holding'] if row and row['holding'] else 0
```

```javascript
const getRemainingQuantity = () => {
  const direction = executeStepForm.value.trade_direction
  const planned = executeStepForm.value.planned_quantity
  const holding = executeStepForm.value.holding_quantity || 0
  
  if (direction === '建仓' || direction === '加仓') {
    return planned  // 建仓/加仓：剩余数量 = 计划数量
  } else if (direction === '减仓' || direction === '清仓') {
    return Math.min(planned, holding)  // 减仓/清仓：剩余数量 = min(计划数量, 持仓数量)
  }
  return planned
}
```

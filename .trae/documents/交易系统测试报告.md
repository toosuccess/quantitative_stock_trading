## 问题发现与修复

### 问题1：账户ID不一致 ✅ 已修复
**现象**：
- 交易记录中的account_id是"ACC001"
- 账户表中的account_id是"ACCBBF480B9"和"ACC20D37714"
- 导致账户更新逻辑找不到正确的账户

**修复方案**：
1. 后端：修改execute_step方法，使用步骤中的account_id
2. 前端：修改TradeExecution.vue，传递正确的account_id

**修复代码**：
```python
# 后端
actual_account_id = account_id or step.get('account_id')
```

```javascript
// 前端
executeStepForm.value.account_id = row.account_id
```

### 问题2：账户金额初始化问题 ✅ 已修复
**现象**：
- 账户的total_assets是100000
- 但available_cash和market_value都是0
- 初始数据不一致

**修复方案**：
创建fix_account_balance.py脚本，初始化账户金额

**修复结果**：
```
东吴证券: available_cash=100000, market_value=0
中信证券: available_cash=150000, market_value=0
```

### 问题3：前端硬编码account_id ✅ 已修复
**现象**：
- 前端代码中多处硬编码'ACC001'
- 导致API调用时传递错误的account_id

**修复方案**：
修改前端代码，使用步骤或计划中的account_id

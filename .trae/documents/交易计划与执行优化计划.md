# 交易计划与执行优化计划

## 需求分析

### 1. 买入数量改成持仓数量
- 前端TradePlan.vue中，将"买入数量"列改为"持仓数量"
- 后端返回holding_quantity（买入数量 - 卖出数量）

### 2. 买入金额改成持仓金额
- 前端TradePlan.vue中，将"买入金额"列改为"持仓金额"
- 后端返回holding_amount（买入金额 - 卖出金额）

### 3. 添加执行计划的加仓、减仓步骤独立
- 通过"添加执行计划"添加的步骤，每个步骤独立记录
- 不合并相同方向的步骤

### 4. 已执行记录状态保持
- 已执行的步骤状态应为"已完成"
- 页面刷新后状态不能恢复成"待执行"
- 检查数据库更新和查询逻辑

### 5. 更新账户金额
- 买入时：可用现金减少，市值增加
- 卖出时：可用现金增加，市值减少
- 需要在execute_step中更新账户信息

#### 2. 修改execute_step更新账户信息
```python
def execute_step(self, step_id: str, trade_price: float, trade_quantity: int, account_id: str = 'ACC001') -> Dict:
    # ... 现有逻辑 ...
    
    # 更新账户信息
    trade_amount = trade_price * trade_quantity
    account = self.get_account(account_id)
    
    if trade_type == '买入':
        # 买入：可用现金减少，市值增加
        new_available_cash = account['available_cash'] - trade_amount
        new_market_value = account['market_value'] + trade_amount
    else:
        # 卖出：可用现金增加，市值减少
        new_available_cash = account['available_cash'] + trade_amount
        new_market_value = account['market_value'] - trade_amount
    
    # 计算总资产
    new_total_assets = new_available_cash + new_market_value
    
    self.update_account(account_id, {
        'available_cash': new_available_cash,
        'market_value': new_market_value,
        'total_assets': new_total_assets
    })
```

#### 3. 添加get_account方法（如果不存在）
```python
def get_account(self, account_id: str) -> Optional[Dict]:
    """获取账户信息"""
    cursor = self.conn.cursor()
    cursor.execute('SELECT * FROM account WHERE account_id = ?', (account_id,))
    row = cursor.fetchone()
    return dict(row) if row else None
```

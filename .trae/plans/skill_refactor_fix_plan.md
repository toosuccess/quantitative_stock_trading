# 技能重构问题分析与修复计划

## 一、问题分析

### 1.1 技术面评分不一致

**旧技能** (`stock_selection_skill.py`) 的技术面评分有5个详细指标：

| 指标 | 满分 | 详细条件 |
|------|------|----------|
| 均线系统 | 25分 | 股价站稳20日均线 + M20向上 + 均线多头排列 |
| 成交量 | 25分 | 成交量>MA5 + 成交量>MA60 + 换手率5%-20% |
| 趋势指标 | 20分 | MACD金叉 + MACD>0 + DMI的+DI>-DI |
| 资金指标 | 15分 | OBV>MAOBV + OBV持续上行 |
| 布林线 | 15分 | 股价在中轨和上轨之间 + 未超买 |

**新技能** (`stock_evaluator_skills.py`) 的技术面评分过于简化：

| 指标 | 满分 | 问题 |
|------|------|------|
| 均线系统 | 25分 | ❌ 只检查股价>MA20，缺少M20向上条件 |
| 成交量 | 25分 | ❌ 只检查volume>0，缺少MA5/MA60/换手率条件 |
| 趋势指标 | 20分 | ❌ 只检查DIFF>DEA，缺少DMI条件 |
| 资金指标 | 15分 | ❌ 用布林带中轨判断，逻辑错误 |
| 布林线 | 15分 | ✅ 基本正确 |

### 1.2 前端显示问题

前端只显示一个指标（均线系统25分），是因为新技能保存的技术明细数据格式不完整。

## 二、修复方案

### 方案1：修复新技能的技术面评分逻辑

需要修改 `stock_evaluator_skills.py` 中的 `calculate_technical_score` 方法，使其与旧技能保持一致。

### 方案2：保留旧技能，删除新技能

如果修复工作量太大，可以保留旧技能，删除新技能。

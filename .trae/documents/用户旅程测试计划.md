# 用户旅程端到端测试计划

## 测试目标
按照真实用户操作流程，从页面模拟完成完整的交易系统操作流程。

## 用户旅程流程

### 旅程1：选股 → 创建交易计划 → 执行交易

```
首页 → 点击"执行选股" → 查看选股结果 → 选择股票 → 创建交易计划 → 填写计划详情 → 保存计划
```

### 旅程2：查看评分 → 分析技术指标

```
首页 → 评分结果页 → 查看股票评分详情 → 分析技术指标数据
```

### 旅程3：交易执行 → 记录交易

```
交易计划页 → 选择待执行计划 → 执行交易 → 记录交易详情 → 更新计划状态
```

### 旅程4：复盘分析 → 查看统计

```
复盘分析页 → 查看胜率统计 → 查看收益曲线 → 查看复盘记录 → 创建复盘记录
```

## 测试步骤详情

### 步骤1：首页导航测试
1. 打开 http://localhost:3000
2. 验证页面加载正常
3. 验证统计卡片数据
4. 点击"执行选股"按钮

1. 验证选股结果列表
2. 查看股票技术指标标签
3. 点击第一只股票的"创建计划"按钮

### 步骤3：交易计划创建测试
1. 点击"新建计划"按钮
2. 填写计划表单：
   - 计划名称：测试买入华胜天成
   - 股票代码：600410
   - 股票名称：华胜天成
   - 计划数量：1000
   - 止损价格：8.50
   - 止盈价格：11.00
3. 点击保存按钮
4. 验证计划创建成功

### 步骤4：评分结果页测试
1. 导航到评分结果页
2. 验证评分详情显示
3. 验证技术指标数据

### 步骤5：复盘分析页测试
1. 导航到复盘分析页
2. 验证统计数据显示
3. 验证收益曲线图表
4. 验证胜率分布饼图

### 步骤6：交易执行页测试
1. 导航到交易执行页
2. 验证页面正常显示

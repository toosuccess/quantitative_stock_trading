# 个人专属交易系统设计方案

## 1. 核心目标

打造一个 **"去情绪化、可量化、可追溯、可重复、可迭代"** 的交易系统，通过标准化流程解决炒股核心痛点，最终实现稳定盈利。

## 2. 核心功能模块拆解

### 2.1 选股模块含评分模块

* 我在trae中开发了"选股技能.md"，并在其中定义了我的选股策略。我会定期运行该技能，根据策略筛选符合要求的股票，包含打分，存入数据库。

### 2.2 账号管理模块

* **多账户支持**：支持创建多个证券账户（实盘/模拟）

* **资产统计**：显示总资产、可用资金、市值、盈亏

* **盈亏计算**：盈亏 = 当前总资产 - 初始资产

* **账户操作**：创建、编辑、删除账户

### 2.3 交易计划 (Strategic Planning)

* **标准化表单**：入场必须满足"计划先行"。

  * **入场逻辑**：记录买入理由（基于评分结果）。

  * **仓位管理**：根据个股波动率（ATR）自动计算建议仓位。

  * **退出策略**：明确止损价（Hard Stop）、止盈价、时间止损（如 3 日不创新高即撤离）。

* **执行步骤**：支持建仓、加仓、减仓、止损清仓、止盈清仓等操作

* **状态管理**：待执行、执行中、已完成、已取消

### 2.4 交易执行模块

* **手动执行**：界面提供操作入口，可以手工执行交易计划

* **步骤执行**：支持按步骤执行交易，记录成交价格和数量

* **交易记录**：完整记录每笔交易的详细信息

* **数量限制**：

  * 加仓时：不能超过剩余数量

  * 减仓时：不能超过持仓数量

* **价格默认值**：

  * 止损清仓/止盈清仓：默认使用目标价格

### 2.5 复盘模块 (Feedback & Iteration)

* **记录与匹配**：自动将实际成交数据与预设交易计划进行匹配。

* **情绪监控**：用户需在成交后记录当时的心理状态（如：贪婪、焦躁、平稳）。

* **失败分类**：系统引导用户将亏损归因为：策略失效、执行不力、突发利空等。

* **统计看板**：胜率、盈亏比、最大回撤、夏普比率可视化。

### 2.6 首页仪表盘

* **统计卡片**：

  * 评分股票数量

  * 待执行交易计划数量

  * 交易胜率

  * 总盈亏

* **交易汇总**：

  * 总买入金额、买入次数

  * 总卖出金额、卖出次数

  * 净盈亏

  * 已完成计划数

* **资产曲线**：可视化资产变化趋势

### 2.7 模拟操作 (Sandbox Environment)

* **虚拟账户**：支持多策略同时运行（A/B Test）。

* **模拟撮合**：模拟真实买卖单、手续费、滑点。

* **策略回测**：支持对选股和评分模型进行历史回测，通过数据支撑信心。

## 4. 数据库设计

### 4.1 数据表清单

| 表名                     | 说明      |
| ---------------------- | ------- |
| account\_info          | 账户信息表   |
| stock\_basic\_info     | 股票基本信息表 |
| score\_record          | 评分记录表   |
| trade\_plan            | 交易计划表   |
| trade\_execution\_step | 交易执行步骤表 |
| trade\_record          | 交易记录表   |
| review\_record         | 复盘记录表   |

### 4.2 关键字段说明

#### account\_info（账户信息表）

* `initial_assets`：初始资产（用于计算盈亏）

* `profit_loss`：盈亏 = 总资产 - 初始资产

#### trade\_plan（交易计划表）

* `status`：pending（待执行）、executing（执行中）、completed（已完成）、cancelled（已取消）

* `holding_quantity`：持仓数量

* `profit`：盈利情况

#### trade\_execution\_step（交易执行步骤表）

* `trade_direction`：建仓、加仓、减仓、止损清仓、止盈清仓

* `target_price`：目标价格

* `executed_quantity`：已执行数量

#### trade\_record（交易记录表）

* `trade_type`：买入、卖出

* `trade_amount`：成交金额

## 7. 业务规则

### 7.1 盈亏计算

* 账户盈亏 = 当前总资产 - 初始资产

* 交易记录盈亏 = 卖出总额 - 买入总额

### 7.2 数量限制

* 加仓：最大数量 = 计划数量 - 持仓数量

* 减仓：最大数量 = 持仓数量

### 7.3 状态流转

* 交易计划：pending → executing → completed/cancelled

* 执行步骤：pending → completed

### 7.4 价格默认值

* 止损清仓/止盈清仓：默认使用目标价格

* 其他操作：默认使用当前价格

# 评分明细显示问题修复计划

## 一、问题分析

### 1. 评分详情页面各维度明细为空
**原因**：数据库中 `technical_detail`、`fundamental_detail` 等字段为空
- 之前的选股脚本 `run_stock_selection.py` 没有保存明细数据
- 只保存了评分结果，没有保存评分明细JSON

### 2. 股票池列表各维度悬浮显示为空
**原因**：同上，数据库中没有明细数据

### 3. 操作列按钮换行
**原因**：按钮宽度不够，需要调整样式

## 二、解决方案

### 方案1：修复选股脚本保存明细数据
修改 `stock_selection_skill.py` 中的保存逻辑，添加明细数据保存

### 方案2：重新评分生成明细数据
使用 `stock_scoring_skill.py` 重新评分，会自动生成明细数据

### 方案3：修复操作列按钮样式
调整按钮样式，确保不换行

## 三、任务拆分

| 序号 | 任务 | 说明 |
|------|------|------|
| 1 | 修复操作列按钮样式 | 设置按钮不换行 |
| 2 | 重新评分生成明细数据 | 使用stock_scoring_skill重新评分 |
| 3 | 验证评分详情页面 | 确认明细数据显示正常 |
| 4 | 验证股票池列表悬浮 | 确认悬浮显示明细正常 |

## 四、具体实现

### 1. 操作列按钮样式
```css
.el-button {
  white-space: nowrap;
}
```

### 2. 重新评分
对现有股票重新评分，生成完整的明细数据

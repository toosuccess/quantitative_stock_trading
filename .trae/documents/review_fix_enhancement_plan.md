# 股票复审技能修复与增强计划

## 📋 需求理解
1. **调整评分权重**：确保各指标之和不超过100分
2. **优化复审意见显示**：符合项打勾 ✓，不符合项打叉 ✗，高亮显示
3. **前端展示增强**：添加"符合得分"单独一列，支持排序

| 指标 | 当前权重 | 问题 |
|------|---------|------|

| K线风控 | 20分 | - |

## 🛠️ 修复方案

### 方案一：调整评分权重（推荐）
| 指标 | 调整后权重 | 说明 |
|------|----------|------|
| MACD核心 | 35分 | 保持核心地位，略降低 |
| 20日均线 | 20分 | 减少5分 |
| K线风控 | 15分 | 减少5分 |
| 布林线 | 10分 | 减少5分 |
| DMI指标 | 10分 | 大幅减少，避免重复 |
| 板块加分 | ±10分 | 保持不变 |
| **总计** | **100分** | ✅ 符合要求 |

### 方案二：计算后归一化
保持各指标原权重，最后计算结果按比例缩放到100分制

## 📝 详细修复步骤

### Step 1：调整评分权重
**修改文件**：`skills/stock_review_skills.py`

1. **MACD评分**（40分 → 35分）
   - 各项子指标同比例调整
   - 保持评分逻辑不变

2. **20日均线评分**（25分 → 20分）
   - 各项子指标同比例调整

3. **K线风控评分**（20分 → 15分）
   - 调整基础分和扣分项

4. **布林线评分**（15分 → 10分）
   - 调整评分档位

5. **DMI评分**（20分 → 10分）
   - 大幅降低权重，作为辅助确认

### Step 2：增强复审意见格式
**修改文件**：`skills/stock_review_skills.py`

1. 为每个评分项添加 `passed` 字段，标记是否符合要求
2. 修改意见生成函数：
   - 符合项：`✓ [绿色] 指标名称: 详情`
   - 不符合项：`✗ [红色] 指标名称: 详情`
3. 保存完整的评分明细到 `review_detail` 字段

### Step 3：后端接口修改
**修改文件**：`backend/app/api/routes.py`

1. 在 `get_stock_pool_scores` 和 `get_review_list` 接口中
2. 返回每个股票的符合得分（符合要求的分数之和）
3. 返回评分明细，供前端使用

### Step 4：前端展示增强
**修改文件**：`frontend/src/views/StockSelection.vue`

1. 添加"符合得分"列，显示符合要求的分数总和
2. 支持该列排序
3. 优化复审信息列的显示格式
4. 点击复审信息时，弹窗显示带有 ✓✗ 标识的详细报告

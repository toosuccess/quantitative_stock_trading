# 交易执行优化计划

## 需求分析

### 1. 添加加仓和减仓步骤

* 当前状态：执行计划时只创建建仓、止损清仓、止盈清仓步骤

* 需求：执行时，自动初始化加仓和减仓步骤

### 2. 执行步骤保存数据库

* 当前状态：点击执行只是创建交易记录，没有更新执行步骤

* 需求：

  * 保存交易记录到数据库

  * 更新执行步骤状态

  * 更新已执行数量

  * 更新交易计划的实际数量

***

## 实施方案

### 一、添加加仓/减仓步骤功能

#### 后端API

* 新增 `POST /api/v1/trade/step` - 创建执行步骤

  * 支持，自动添加加仓、减仓步骤

  * 也支持，手工创建添加

#### 前端实现

* TradeExecution.vue添加"添加步骤"按钮

* 弹出表单选择步骤类型（加仓/减仓）、目标价格、数量

### 二、执行步骤保存逻辑

#### 执行流程

```
1. 点击执行按钮
2. 弹出执行表单（价格、数量）
3. 确认执行：
   - 创建交易记录（trade_record）
   - 更新执行步骤状态和数量（trade_execution_step）
   - 更新交易计划实际数量（trade_plan.actual_quantity）
   - 检查是否全部执行完成，更新步骤状态
```

#### 后端API调整

* 修改 `POST /api/v1/trade/record` - 创建交易记录时同步更新步骤和计划

* 新增 `PUT /api/v1/trade/step/{step_id}` - 更新执行步骤

#### 数据更新逻辑

```python
# 创建交易记录时
1. 保存交易记录到 trade_record
2. 更新执行步骤：
   - executed_quantity += trade_quantity
   - 如果 executed_quantity >= planned_quantity:
     - status = 'completed'
3. 更新交易计划：
   - actual_quantity += trade_quantity
   - 检查所有步骤是否完成：
     - 如果全部完成: status = 'completed'
```

## 预期效果

### 添加步骤功能

```
交易执行页面

[添加步骤] 按钮

弹出表单：
- 计划编号: PLAN_xxx (下拉选择)
- 步骤类型: 加仓/减仓
- 目标价格: 输入框
- 计划数量: 输入框
- 原因: 输入框

[确认] [取消]
```

### 执行步骤逻辑

```
执行前：
步骤: 建仓 | 计划: 1000 | 已执行: 0 | 状态: 待执行

点击执行，输入：价格 9.50，数量 500

执行后：
步骤: 建仓 | 计划: 1000 | 已执行: 500 | 状态: 待执行
交易记录: 买入 500股 @ 9.50

再次执行，输入：价格 9.60，数量 500

执行后：
步骤: 建仓 | 计划: 1000 | 已执行: 1000 | 状态: 已完成
交易记录: 买入 500股 @ 9.60
计划: actual_quantity = 1000
```

### 状态更新规则

```
执行步骤状态：
- pending: 待执行
- completed: 已执行数量 >= 计划数量

交易计划状态：
- executing: 有步骤在执行中
- completed: 所有步骤都已完成
```

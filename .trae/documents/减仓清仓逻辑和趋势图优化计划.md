# 减仓清仓逻辑和趋势图优化计划

## 需求分析

### 需求1：减仓清仓逻辑
**当前情况**：
- 执行减仓操作时，没有判断减仓数量是否等于持仓数量
- 只有执行"止损清仓"、"清仓"、"止盈清仓"时才将计划标记为已完成

**期望**：
- 执行减仓操作时，如果减仓数量 = 持仓数量，代表清仓了
- 交易计划应标记为已完成

### 需求2：趋势图X轴按时间顺序
**当前情况**：
- 趋势图X轴使用 `.reverse()` 反转了数据顺序
- 可能导致时间顺序不正确

**期望**：
- X轴按时间顺序（从早到晚）显示

---

## 实施方案

### 1. 修改减仓清仓逻辑

文件：`backend/app/services/trading_manager.py`

修改 `execute_step` 方法：
```python
# 原来：只有止损清仓、清仓、止盈清仓才标记计划完成
if step['trade_direction'] in ['止损清仓', '清仓', '止盈清仓']:
    self.update_trade_plan(step['plan_id'], {'status': 'completed'})

# 修改为：减仓时判断是否清仓
if step['trade_direction'] in ['止损清仓', '清仓', '止盈清仓']:
    self.update_trade_plan(step['plan_id'], {'status': 'completed'})
elif step['trade_direction'] == '减仓':
    # 判断减仓后是否清仓
    plan = self.get_trade_plan(step['plan_id'])
    if plan:
        new_holding = plan['holding_quantity'] - trade_quantity
        if new_holding <= 0:
            # 清仓了，标记计划完成
            self.update_trade_plan(step['plan_id'], {'status': 'completed', 'holding_quantity': 0})
        else:
            self.update_trade_plan(step['plan_id'], {'holding_quantity': new_holding})
```

### 2. 修改趋势图X轴顺序

文件：`frontend/src/views/ScoreDetail.vue`

修改 `renderIndicatorChart` 方法：
```javascript
// 原来：使用 .reverse() 反转
const times = scoreHistory.value.map(s => s.create_time || s.score_date).reverse()

// 修改为：按时间正序（从早到晚）
const times = scoreHistory.value.map(s => s.create_time || s.score_date)
// 如果数据是倒序的，先排序
times.sort()
```

# 架构调整计划

## 需求背景

根据需求文档，整体架构需要调整：
1. **选股技能**是另一个项目的能力，会在后台独立运行，跑完后写入数据库
2. **当前项目边界**：读取数据库中已评分的股票，在页面上呈现，为用户旅程提供页面操作

## 架构调整方案

### 1. 后端API调整

#### 需要保留的API（读取数据库数据）
| API | 方法 | 说明 |
|-----|------|------|
| GET /api/v1/stocks/pool | GET | 获取行业股票池 |
| GET /api/v1/trade/plans | GET | 获取交易计划列表 |
| POST /api/v1/trade/plan | POST | 创建交易计划 |
| GET /api/v1/trade/statistics/{id} | GET | 获取交易统计 |

#### 需要新增的API（读取评分记录）
| API | 方法 | 说明 |
|-----|------|------|
| GET /api/v1/scores | GET | 获取评分记录列表（从score_record表读取） |
| GET /api/v1/scores/{code} | GET | 获取指定股票的评分详情 |

#### 需要移除/修改的API
| API | 处理方式 |
|-----|----------|
| GET /api/v1/select | 移除（选股是另一个项目） |
| GET /api/v1/stocks/score/{code} | 修改为从数据库读取 |
| GET /api/v1/stocks/realtime/{code} | 保留（可选，用于实时行情） |

### 2. 前端页面调整

#### Home.vue
- 从API获取统计数据（交易计划数、胜率等）
- 移除"执行选股"按钮的选股调用
- 改为跳转到评分结果页查看已评分股票

#### StockSelection.vue
- 改名为 ScoreList.vue 或保持原名
- 从 `/api/v1/scores` 获取数据库中的评分记录
- 移除实时选股逻辑

#### TradePlan.vue
- 保持现有逻辑，从API获取交易计划
- 创建计划时保存到数据库

### 3. 数据库表结构

使用现有的 `score_record` 表存储评分结果：
```sql
-- score_record 表已存在，包含以下字段：
-- stock_code, stock_name, score_date, total_score, rating
-- ma_score, macd_score, rsi_score, bollinger_score, volume_score, obv_score
-- close_price, volume, 等技术指标数据
```

## 实施步骤

### 步骤1：后端API修改
1. 新增 `/api/v1/scores` 接口，从 score_record 表读取数据
2. 修改 `/api/v1/stocks/score/{code}` 接口，从数据库读取而非实时计算
3. 移除或注释 `/api/v1/select` 选股接口

### 步骤2：前端页面修改
1. 修改 Home.vue，从新API获取统计数据
2. 修改 StockSelection.vue，从 `/api/v1/scores` 获取评分记录
3. 更新 api/index.js 接口定义

### 步骤3：测试验证
1. 验证数据库中的评分记录能正确显示在页面上
2. 验证交易计划功能正常
3. 完成用户旅程测试

### 步骤4：输出测试报告
1. 生成架构调整后的测试报告
2. 记录所有修改内容

## 预期结果

1. 前端页面展示的是数据库中已存储的评分记录
2. 不再调用实时选股功能
3. 用户可以查看评分结果、创建交易计划
4. 数据一致性：页面数据与数据库数据一致

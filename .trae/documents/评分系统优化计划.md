# 评分系统优化计划

## 一、需求分析

### 1. 添加综合评价功能
- 在股票池列表和评分详情页面添加本次评分的综合评价
- 综合评价需要基于各维度评分自动生成文字描述

### 2. 数据一致性问题分析与解决

#### 问题1：股票代码格式不一致
**现象**：数据库中存在 `688017` 和 `sh688017` 两种格式的股票代码
**原因**：评分时传入的股票代码格式不统一
**影响**：
- 股票池列表API使用 `sr2.stock_code = sr1.stock_code` 匹配，导致不同格式的代码被当作不同股票
- 同一只股票可能有多条"最新"记录
- 评分详情页面可能获取到错误的记录

#### 问题2：评分次数统计不准确
**现象**：股票池列表显示的评分次数与实际不符
**原因**：股票代码格式不一致导致COUNT统计错误

#### 问题3：股票名称不一致
**现象**：数据库中同一股票代码对应不同的股票名称（如 `002065` 和 `东华软件`）
**原因**：评分时传入的股票名称来自不同来源

## 二、解决方案

### 步骤1：数据库修改
1. 添加 `summary` 字段存储综合评价
2. 统一股票代码格式（去掉前缀，只保留纯代码）

### 步骤2：后端修改
1. 修改 `stock_scoring_skill.py`：
   - 统一股票代码格式（去掉 sh/sz 前缀）
   - 添加综合评价生成函数
   - 保存综合评价到数据库

2. 修改 `routes.py`：
   - 修改 `/stocks/pool/scores` API 返回综合评价
   - 修改 `/scores/{stock_code}` API 匹配逻辑

### 步骤3：前端修改
1. 修改 `StockSelection.vue`：
   - 添加综合评价列
   - 显示综合评价文字

2. 修改 `ScoreDetail.vue`：
   - 添加综合评价展示区域
   - 显示本次评分的综合分析

## 四、综合评价生成规则

根据各维度评分生成综合评价：

```
综合评价 = {
  技术面评价: 根据技术面得分生成（优秀/良好/一般/较差）
  基本面评价: 根据基本面得分生成
  消息面评价: 根据消息面得分和事件生成
  政策面评价: 根据政策面得分和政策生成
  风险提示: 根据减项扣分生成
  投资建议: 综合各维度给出建议
}
```

## 五、验收标准

1. 股票池列表显示综合评价
2. 评分详情页面显示综合评价
3. 股票代码格式统一（纯数字或纯代码）
4. 股票池列表和评分详情页数据一致
5. 评分次数统计准确

# 选股和评分技能数据库写入逻辑修正计划

## 需求分析

### 当前问题

1. **选股技能**：没有写入数据库的功能，选股结果只在内存中
2. **评分技能**：从stock\_basic\_info读取股票池，但该表为空
3. **表关系不清晰**：stock\_basic\_info和score\_record应该是主从关系

### 期望

1. **选股技能**：推荐股票时，先写入stock\_basic\_info，然后写入score\_record
2. **评分技能**：从stock\_basic\_info读取股票，每次评分新增score\_record
3. **表关系**：stock\_basic\_info是主表，score\_record是从表，一个股票可以有多个评分记录

***

## 数据库表关系

### 主表：stock\_basic\_info

* 存储股票基本信息

* 每只股票一条记录

* 字段：stock\_code, stock\_name, exchange, industry, sector, status等

### 从表：score\_record

* 存储评分记录

* 每次评分新增一条记录

* 字段：stock\_code, score\_date, total\_score, rating等

* 外键：stock\_code关联stock\_basic\_info.stock\_code

***

## 实施方案

### 1. 修改选股技能

文件：`skills/stock_selection_skill.py`

添加数据库写入功能：

```python
def save_to_database(self, selected_stocks):
    """
    保存选股结果到数据库
    
    1. 先写入stock_basic_info（如果不存在）
    2. 再写入score_record（每次选股新增一条）
    """
    for stock in selected_stocks:
        # 1. 写入stock_basic_info（如果不存在）
        self.save_stock_basic_info(stock)
        
        # 2. 写入score_record
        self.save_score_record(stock)
```

```python
def get_stock_pool(self):
    """
    从stock_basic_info读取股票池
    """
    # 只从stock_basic_info读取，不再从score_record读取
    cursor.execute('''
        SELECT stock_code, stock_name, exchange, industry, sector
        FROM stock_basic_info
        WHERE status = 'normal'
        ORDER BY stock_code
    ''')
```

修改保存逻辑：

```python
def save_score_record(self, stock_code, score_data):
    """
    保存评分记录
    每次评分新增一条记录，不更新已有记录
    """

        INSERT INTO score_record (stock_code, score_date, ...)
```

### 3. 数据迁移

如果stock\_basic\_info为空，需要从score\_record迁移数据：

```python
def migrate_stock_basic_info():
    """
    从score_record迁移股票基本信息到stock_basic_info
    """
    cursor.execute('''
        INSERT OR IGNORE INTO stock_basic_info (stock_code, stock_name)
        SELECT DISTINCT stock_code, stock_name
        FROM score_record
    ''')
```

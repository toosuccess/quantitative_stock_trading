# 评分记录优化计划

## 需求分析

### 需求1：每次打分都要记录
- 当前使用 `INSERT OR REPLACE`，同一天同一股票的评分会被替换
- 需要改为每次打分都新增记录，即使同一天也分别记录
- 需要添加评分时间戳来区分同一天的不同评分

### 需求2：趋势图描述各指标变化
- 当前趋势图只显示总分和收盘价
- 需要显示各指标（均线、MACD、成交量、布林线、OBV）的变化趋势

### 需求3：显示各指标的打分依据
- 当前表格只显示得分
- 需要显示各指标的原值和得分
- 例如：MA20=365.34，得分=25

---

## 实施方案

### 1. 修改评分技能

文件：`skills/stock_scoring_skill.py`

修改保存逻辑：
```python
# 原来：INSERT OR REPLACE（同一天会替换）
# 修改为：INSERT（每次新增记录）
INSERT INTO score_record (...)
VALUES (...)
```

添加评分时间字段：
- 使用 `create_time` 区分同一天的不同评分

### 2. 修改数据库表

添加字段或使用现有字段：
- 使用 `create_time` 作为评分时间戳
- 确保同一天可以有多条记录

### 3. 修改前端趋势图

文件：`frontend/src/views/ScoreDetail.vue`

修改趋势图显示各指标变化：
- X轴：评分时间
- Y轴：得分
- 多条折线：均线、MACD、成交量、布林线、OBV

### 4. 修改前端表格

文件：`frontend/src/views/ScoreDetail.vue`

修改表格显示打分依据：
- 每个指标列显示：原值 + 得分
- 例如：MA20=365.34 (25分)

# 市值计算问题修复计划

## 问题分析

### 当前数据
```
账户信息:
  东吴证券: 总资产=100000, 初始资产=100000, 可用现金=85385, 市值=14615

交易汇总:
  宁德时代: 买入=1000股(365340), 卖出=1000股(350725), 持仓=0股
```

### 问题
- 持仓=0股，但市值=14615
- 市值应该是0

### 原因分析
当前代码逻辑：
```python
if trade_type == '买入':
    new_market_value = account['market_value'] + trade_amount
else:
    new_market_value = max(0, account['market_value'] - trade_amount)
```

问题：
- 买入时：市值 += 买入金额（365340）
- 卖出时：市值 -= 卖出金额（350725）
- 结果：市值 = 365340 - 350725 = 14615

**错误**：卖出时应该减去的是"卖出股票对应的买入成本"，而不是"卖出金额"

### 正确逻辑
市值 = 当前持仓 × 当前价格

或者：
- 买入时：市值 += 买入金额
- 卖出时：市值 -= 卖出股票对应的买入成本

---

## 解决方案

### 方案1：实时计算市值（推荐）
根据当前持仓和当前价格实时计算市值，而不是累计计算。

### 方案2：修复卖出逻辑
卖出时，减去的是卖出股票对应的买入成本。

---

## 实施方案（方案1）

### 1. 修改账户市值计算
在获取账户信息时，根据持仓实时计算市值：
```python
# 计算市值 = 持仓数量 × 当前价格
market_value = holding_quantity × current_price
```

### 2. 修改交易执行逻辑
不再在交易执行时更新市值，而是：
- 买入时：可用现金 -= 买入金额
- 卖出时：可用现金 += 卖出金额
- 市值由持仓和价格实时计算

### 3. 修复现有数据
将东吴证券的市值从14615改为0。

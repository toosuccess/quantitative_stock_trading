# 计划数量验证和盈亏计算修复计划

## 需求分析

### 问题1：计划数量为0时禁止确认添加
**当前情况**：
- 添加执行步骤时，如果计划数量为0，仍可点击"确认添加"

**期望**：
- 计划数量为0时，禁止"确认添加"按钮

### 问题2：盈亏金额计算逻辑检查
**当前情况**：
- 交易计划列表中"盈利情况"显示-24610.00
- 账号管理中盈亏计算：盈亏 = 卖出总额 - 买入总额

**问题分析**：
1. 交易计划盈利计算：
   - 如果持仓为0，盈利 = 卖出金额 - 买入金额（已实现盈利）
   - 如果有持仓，盈利 = 未实现盈利 + 已实现盈利
   - 未实现盈利 = (当前价格 - 买入均价) * 持仓数量

2. 账号管理盈亏计算：
   - 当前：盈亏 = 卖出总额 - 买入总额
   - 问题：这个计算方式不正确
   - 正确方式：盈亏 = 总资产 - 初始资产

**期望**：
- 修正盈亏计算逻辑

### 问题2修复：盈亏计算逻辑

文件：`backend/app/services/trading_manager.py`

账号管理盈亏计算修正：
```python
# 方案1：使用初始资产计算
# 需要在account_info表中添加initial_assets字段

# 方案2：使用当前市值计算
# 盈亏 = 当前总资产 - 初始投入
# 初始投入 = 买入总额 - 卖出总额（净投入）
# 盈亏 = 当前市值 - 净投入

# 方案3：使用已实现盈亏 + 未实现盈亏
# 已实现盈亏 = 卖出总额 - 卖出数量 * 买入均价
# 未实现盈亏 = (当前价格 - 买入均价) * 持仓数量
```

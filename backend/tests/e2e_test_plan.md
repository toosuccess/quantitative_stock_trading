# 个人专属交易系统端到端测试方案

## 1. 测试目标与范围

### 1.1 测试目标
验证个人专属交易系统的核心功能模块是否按预期工作，确保：
- 数据获取模块能正确获取实时股价和技术指标
- 选股评分模块能筛选出符合条件的股票（至少5只）
- 交易管理模块能正确执行CRUD操作
- 前端界面能正确展示数据和响应交互

### 1.2 测试范围

#### 后端API测试
- 数据获取模块：AkShare接口连接、实时行情、K线数据、技术指标计算
- 选股评分模块：选股功能、评分计算、评级生成
- 交易管理模块：交易计划CRUD、执行步骤、交易记录、复盘记录

#### 前端界面测试
- 首页展示：统计数据、资产曲线、快速操作
- 选股结果页面：股票列表、评分展示
- 交易计划管理页面：计划创建、编辑、删除
- 复盘分析页面：交易统计、胜率分析

## 2. 测试用例清单

### 2.1 后端API测试用例

#### TC-API-001: 数据获取模块测试
**测试步骤**：
1. 测试AkShare数据接口连接
2. 获取股票002065的实时行情
3. 获取股票002065的K线数据（60天）
4. 计算技术指标（MA、MACD、RSI、布林线、OBV）

**预期结果**：
- 实时行情包含股票代码、名称、最新价、涨跌幅等字段
- K线数据至少包含20条记录
- 技术指标包含ma5、ma20、diff、dea、rsi、bb_upper、bb_lower等

#### TC-API-002: 选股功能测试
**测试步骤**：
1. 执行选股功能，限制返回5只股票
2. 验证返回股票数量 >= 1
3. 验证每只股票包含stock_code、stock_name、score、tech_result字段
4. 验证技术条件通过数 >= 3

**预期结果**：
- 返回至少1只符合条件的股票
- 每只股票评分在0-100之间
- 技术条件至少通过3项

#### TC-API-003: 评分功能测试
**测试步骤**：
1. 对股票002065进行评分
2. 验证评分结果包含total_score、rating字段
3. 验证评级在['强烈推荐', '推荐', '中性', '观望', '不推荐']范围内

**预期结果**：
- 评分结果完整
- 评级符合预定义值

#### TC-API-004: 交易计划CRUD测试
**测试步骤**：
1. 创建交易计划

## 3. 测试环境

### 3.1 后端环境
- Python 3.8+
- FastAPI
- SQLite数据库
- AkShare数据源

### 3.2 前端环境
- Node.js 16+
- Vue 3
- Element Plus
- ECharts

### 3.3 测试工具
- pytest（后端单元测试）
- Playwright（端到端测试）
- 浏览器：Chromium

## 4. 测试执行计划

### 4.1 执行顺序
1. 运行后端单元测试
2. 启动后端FastAPI服务（端口8000）
3. 启动前端Vue服务（端口5173）
4. 使用Playwright执行端到端测试
5. 生成测试报告

### 4.2 测试数据准备
- 使用测试数据库：test_trading_system.db
- 测试股票代码：002065（东华软件）
- 测试账户ID：TEST001

## 5. 验收标准

### 5.1 后端API测试
- 所有单元测试通过率 >= 90%
- 选股功能必须返回至少1只股票
- 技术指标计算结果准确

### 5.2 前端界面测试
- 所有页面正常加载
- 关键交互功能正常
- 无JavaScript错误

### 5.3 端到端测试
- 完整业务流程可执行
- 数据流转正确
- 界面响应及时

## 6. 风险与改进建议

### 6.1 风险点
- AkShare接口可能因网络问题超时
- 股票数据实时性依赖数据源
- 测试数据库需要定期清理

### 6.2 改进建议
- 增加接口超时重试机制
- 添加数据缓存层
- 实现测试数据自动清理
- 增加性能测试用例
- 添加CI/CD集成测试

# 账户盈亏显示一致性修复计划

## 问题分析
用户反馈账户信息的"盈亏"显示与交易计划明细中的盈亏汇总不一致。

**当前显示位置**:
1. **账户信息区域**（Review.vue 第 58-64 行）：
   - 显示：`总资产 | 盈亏 | 计划数`
   - 数据来源：`account.profit_loss`

2. **交易计划列表底部**（TradePlan.vue 第 119-123 行）：
   - 显示：`合计：持仓金额 | 总盈亏`
   - 数据来源：所有计划的 `profit` 字段汇总

**问题根源**:

### 账户盈亏计算逻辑（后端 trading_manager.py 第 1247-1251 行）
```python
# 盈亏 = 持仓市值 - 买入成本（只计算交易产生的盈亏）
if total_buy_cost > 0:
    account['profit_loss'] = market_value - total_buy_cost
else:
    account['profit_loss'] = 0
```
- **计算方式**：基于实际交易记录（trade_record）
- **公式**：盈亏 = ∑(持仓数量 × 当前股价) - ∑买入金额
- **性质**：实际交易产生的盈亏（已实现 + 未实现）

### 交易计划盈亏计算逻辑（后端 trading_manager.py 第 214-218 行）
```python
unrealized_profit = (current_price - buy_avg_price) * holding_quantity
realized_profit = sell_amount - (sell_quantity * buy_avg_price)
profit = unrealized_profit + realized_profit
```
- **计算方式**：基于交易计划关联的交易记录
- **公式**：盈亏 = 未实现盈亏 + 已实现盈亏
- **性质**：计划维度的账面盈亏

**结论**：两者的计算逻辑和数据来源不同，导致数值不一致。

## 解决方案

### 方案 A：统一使用交易计划的盈亏汇总（推荐）✅
修改 `get_all_accounts` 方法，账户盈亏 = 该账户下所有交易计划的 profit 总和

**优点**：
- 数据一致性高，账户盈亏与计划盈亏汇总一致
- 逻辑清晰，用户容易理解

**修改位置**：
- `trading_manager.py` 的 `get_all_accounts` 方法

### 方案 B：保持现状，但明确标注区别
在 UI 上明确标注：
- 账户盈亏：实际交易盈亏
- 计划盈亏：计划账面盈亏

**优点**：
- 不需要修改后端逻辑
- 用户可以区分两种盈亏

**缺点**：
- 用户可能仍然感到困惑

## 实施步骤（采用方案 A）

### 步骤 1：修改 get_all_accounts 方法
在遍历 holdings 计算盈亏后，改为汇总该账户下所有交易计划的 profit：

```python
# 查询该账户下所有交易计划的盈亏总和
cursor.execute('''
    SELECT COALESCE(SUM(profit), 0) as total_profit
    FROM trade_plan
    WHERE account_id = ?
''', (account['account_id'],))
plan_profit = cursor.fetchone()['total_profit']
account['profit_loss'] = plan_profit
```

### 步骤 2：验证测试
- 重启后端服务
- 查看账户盈亏显示
- 与交易计划列表的总盈亏对比，确认一致

## 预期效果
1. 账户信息的盈亏 = 该账户下所有交易计划的 profit 总和
2. 交易计划列表底部的总盈亏 = 所有计划的 profit 总和
3. 两者完全一致

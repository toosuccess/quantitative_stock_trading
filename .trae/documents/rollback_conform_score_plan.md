# 回滚符合得分功能计划

## 问题分析
用户不希望引入"符合得分"这个新概念，要求只保留原来的"复合得分"（复审得分）单独显示。

## 需要修改的文件

### 1. 前端文件
- **文件**: `frontend/src/views/StockSelection.vue`
- **修改内容**:
  - 移除"符合得分"列
  - 移除 `getConformScoreType` 函数
  - 移除排序逻辑中对符合得分的处理

### 2. 后端文件
- **文件**: `backend/app/api/routes.py`
- **修改内容**:
  - 移除接口中对 `conform_score` 的返回
  - 恢复到之前只返回复审得分的状态

### 3. 复审技能文件
- **文件**: `skills/stock_review_skills.py`
- **修改内容**:
  - 保留计算逻辑（防止数据库已有数据问题），但可以简化
  - 或者完全移除符合得分相关代码

### 4. 数据库
- **字段**: `conform_score` 字段可以保留在数据库中，不需要删除
- **原因**: 防止数据丢失，将来可能需要

## 执行步骤
1. 修改前端 StockSelection.vue，移除符合得分列
2. 修改后端 routes.py，移除符合得分返回
3. 重启服务验证
4. 端到端测试

## 注意事项
- 保留数据库字段，不删除数据
- 确保前端只显示复审得分一列
- 保持代码简洁清晰

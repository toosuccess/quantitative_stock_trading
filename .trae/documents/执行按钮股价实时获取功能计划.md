# 执行按钮股价实时获取功能计划

## 需求分析
当用户点击"执行"按钮时，会弹出交易执行窗口。当前窗口中的股价输入框需要：
1. **实时获取**：自动从后端获取该股票的最新价格
2. **自动填充**：将最新价格填充到股价输入框中
3. **可手动修改**：用户仍可以手动调整价格

## 实施步骤

### 步骤 1：定位相关页面和组件 ✅
- 找到包含"执行"按钮的 Vue 文件：TradeExecution.vue
- 确认弹出窗口的实现方式：el-dialog (executeStepDialogVisible)
- 找到股价输入框的绑定变量：executeStepForm.trade_price (第 199 行)
- 找到打开对话框的函数：showExecuteStepDialog (第 504-540 行)

### 步骤 2：检查后端 API ✅
- 已有获取股票实时价格的 API：`/stocks/realtime/${code}`
- 前端 API 方法：stockApi.getRealtime(code) (api/index.js 第 24 行)
- 返回格式：包含 price 字段

### 步骤 3：修改前端代码
- 在 showExecuteStepDialog 函数中调用 stockApi.getRealtime()
- 将获取到的价格自动填充到 executeStepForm.trade_price
- 添加 loading 状态提示
- 添加错误处理（API 失败时使用原价格）

### 步骤 4：优化用户体验
- 在价格输入框旁添加"刷新"按钮（可选）
- 显示价格获取时间戳
- 添加价格变动提示（如果价格有更新）

### 步骤 5：测试验证
- 前端服务重启验证
- 点击执行按钮测试
- 验证价格是否正确获取和显示
- 测试异常情况（网络错误、股票不存在等）

## 预期效果
1. 点击"执行"按钮后，弹出窗口自动显示最新股价
2. 股价输入框自动填充最新价格
3. 用户可以手动修改价格
4. 如果获取失败，显示错误提示但不影响窗口打开

## 技术要点
- 使用 async/await 处理异步 API 调用
- 使用 loading 状态提升用户体验
- 错误处理要友好，不影响主流程

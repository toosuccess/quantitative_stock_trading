# 账号管理与复盘分析优化计划

## 需求分析

### 1. 账号管理界面
- 一个主账号下可以有多个证券账号
- 记录各证券账号的金额情况（总资产、可用资金、市值、盈亏等）
- 支持添加、编辑、删除证券账号

### 2. 交易计划关联证券账号
- 创建交易计划时选择证券账号
- 交易计划列表显示关联的证券账号
- 按证券账号筛选交易计划

### 3. 复盘分析层级结构
- 按证券账号 → 交易计划 → 交易记录 组织信息
- 支持层级展开查看
- 显示各层级的统计数据

### 4. 复盘分析统计报表
- 对接真实数据，替换mock数据
- 计算真实的胜率、盈亏比、最大回撤等

---

## 实施方案

### 一、账号管理功能

#### 数据库调整
- 已有account_info表，包含：account_id, account_name, account_type, broker, total_assets, available_cash, market_value, profit_loss等字段
- 需要添加主账号关联字段（parent_account_id）

#### 后端API
- `GET /api/v1/accounts` - 获取账号列表
- `POST /api/v1/account` - 创建账号
- `PUT /api/v1/account/{account_id}` - 更新账号
- `DELETE /api/v1/account/{account_id}` - 删除账号
- `GET /api/v1/account/{account_id}/summary` - 获取账号汇总信息

#### 前端实现
- 新增AccountManagement.vue页面
- 显示账号列表（账号名称、类型、券商、总资产、可用资金、市值、盈亏）
- 添加/编辑账号对话框

### 二、交易计划关联证券账号

#### 后端调整
- 修改创建计划接口，必须选择account_id
- 修改获取计划列表接口，支持按account_id筛选

#### 前端调整
- 创建计划时选择证券账号（下拉框）
- 交易计划列表显示证券账号名称
- 添加按账号筛选功能

### 三、复盘分析层级结构

#### 页面结构
```
复盘分析页面
├── 账号选择（下拉框）
├── 统计看板（胜率、盈亏比、最大回撤、交易次数）
├── 层级列表
│   ├── 证券账号卡片
│   │   ├── 账号名称、总资产、盈亏
│   │   └── 交易计划列表（可展开）
│   │       ├── 计划名称、股票、盈亏
│   │       └── 交易记录列表（可展开）
│   │           └── 记录详情
```

#### 后端API
- `GET /api/v1/review/accounts` - 获取有交易记录的账号列表
- `GET /api/v1/review/account/{account_id}/plans` - 获取账号下的交易计划
- `GET /api/v1/review/plan/{plan_id}/records` - 获取计划下的交易记录

### 四、统计报表真实数据

#### 后端API调整
- 修改 `GET /api/v1/trade/statistics/{account_id}` 返回真实统计数据
- 计算逻辑：
  - 胜率 = 盈利交易次数 / 总交易次数
  - 盈亏比 = 平均盈利金额 / 平均亏损金额
  - 最大回撤 = 历史最高点后的最大跌幅
  - 总交易次数 = 交易记录总数

### 统计数据来源
```
胜率计算:
- 从trade_record获取所有卖出记录
- 计算盈利次数 / 总交易次数

盈亏比计算:
- 计算所有盈利交易的平均盈利金额
- 计算所有亏损交易的平均亏损金额
- 比值 = 平均盈利 / 平均亏损

最大回撤计算:
- 按日期排序计算累计收益
- 找到历史最高点后的最大跌幅
```

# 执行步骤状态与显示优化计划

## 需求分析

### 问题1：执行后状态还是"待执行"
**当前逻辑**：
```python
step_status = 'completed' if new_executed >= step['planned_quantity'] else 'pending'
```
- 只有当已执行数量 >= 计划数量时，状态才会变成"completed"
- 用户期望：执行一次后状态就变成"已完成"

**期望逻辑**：
- 执行一次后，状态应该变成"已完成"
- 每个步骤只能执行一次

### 问题2：加仓步骤不要汇总
**当前情况**：
- 前端显示执行步骤列表
- 每个步骤独立显示

**需要确认**：
- 用户可能指的是交易记录列表中的汇总显示
- 或者是其他地方的汇总逻辑

---

## 实施方案

### 问题1修复：执行后状态变成"已完成"

#### 后端修改
文件：`backend/app/services/trading_manager.py`

```python
# 修改前
step_status = 'completed' if new_executed >= step['planned_quantity'] else 'pending'

# 修改后
step_status = 'completed'  # 执行一次后直接变成已完成
```

### 问题2修复：加仓步骤独立显示

#### 前端修改
文件：`frontend/src/views/TradeExecution.vue`

检查是否有汇总逻辑，如果有则移除。
